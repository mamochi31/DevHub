# useState について

Reactの関数コンポーネントで**「状態（State）」**を管理するための最も基本的なフックです。

## なぜ必要なのか？（通常の変数との違い）

通常のJavaScriptの変数（例：`let count = 0`）は、値を変更しても画面の表示は変わりません。
`useState` を使うことで、値が更新されたときにReactが自動的にコンポーネントを**再レンダリング（画面の再描画）**し、最新のデータを画面に反映してくれます。

## 基本的な構文

```typescript
const [state, setState] = useState(initialState);
```

- **`state`**: 現在の状態（値）が入っている変数です。
- **`setState`**: 状態を更新するための関数（セッター関数）です。この関数を呼び出すことで値が変わり、画面が再描画されます。
- **`initialState`**: 状態の初期値です（数値、文字列、配列、オブジェクトなど何でも指定できます）。

---

## 💡 補足：useState の重要な注意点

### 1. 状態（State）は直接変更してはいけない
Stateの値を直接書き換えてもReactは変更を検知できず、画面が更新されません。必ずセッター関数を使います。
```typescript
// ❌ 悪い例（直接代入）
count = count + 1; 

// ⭕ 良い例（セッター関数を使う）
setCount(count + 1);
```

### 2. 前の状態に依存する更新（コールバック関数を渡す）
現在の値をもとに新しい値を計算する場合、セッター関数の中に「関数」を渡す書き方が安全です。
ReactのState更新は非同期で行われることがあるため、この書き方をすることで「古い値を参照してしまうバグ」を防げます。
```typescript
// prev（前の値）を受け取り、それに+1したものを返す
setCount((prev) => prev + 1);
```

### 3. 遅延初期化（Lazy Initialization）
初期値の計算に時間がかかる場合（例：`localStorage`からの読み込みや重いループ処理など）、`useState`の引数に**関数**を渡すことで、**コンポーネントの初回表示時（マウント時の1回）だけ**計算処理を実行させることができます。
```typescript
// 関数を渡すことで、毎回の再レンダリング時にローカルストレージを見に行く無駄を防げる
const [theme, setTheme] = useState(() => {
  const savedTheme = localStorage.getItem("theme");
  return savedTheme || "light";
});
```

### 4. オブジェクトや配列の更新
Stateがオブジェクトや配列の場合、直接その中身を変更してはいけません。必ず新しいオブジェクトや配列を作成して、セッター関数に渡す必要があります（**イミュータブルな更新**）。
スプレッド構文 `...` を使うのが一般的です。

```typescript
// ❌ 悪い例: 直接プロパティを変更
user.name = 'New Name';
setUser(user);

// ⭕ 良い例: 新しいオブジェクトを作成して更新
setUser(prevUser => ({ ...prevUser, name: 'New Name' }));
```
