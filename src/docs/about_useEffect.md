# useEffectについて

関数コンポーネントにおいて**「副作用（Side Effects）」**を扱うためのフックです。

- **第1引数 (setup)**: 副作用を含む関数（コールバック関数）を渡します。
- **第2引数 (dependencies / 依存配列)**: 省略可能な配列です。この配列の中身によって、第1引数の関数が「いつ実行されるか」をコントロールします。

## 副作用とは
コンポーネントのUIレンダリング（画面の描画）とは直接関係のない処理全般を指します。

## 第2引数（依存配列）のパターン

### 依存配列を渡さない場合
コンポーネントが初めて表示された時（マウント時）と、状態（State）やPropsが変更されて再レンダリングされるたびに毎回実行されます。
無限ループの原因になりやすいため、意図的に使うケースは少なめです。

### 空の配列 `[]` を渡す場合
コンポーネントがマウントされた直後の初回レンダリング時に1度だけ実行されます。
現在の `usePosts.ts` で、モックデータを取得するために使用されているのがまさにこのパターンです。
記事データを何度も取得し直す必要はないため、空の配列を指定しています。

### 配列に特定の変数を渡す場合
初回レンダリング時と、指定した変数（`userId` など）の値が前回のレンダリング時から変化した場合のみ実行されます。
例えば「検索キーワードが変わるたびにAPIを叩き直す」といった要件で活躍します。

## クリーンアップ関数
`useEffect` の第1引数で関数を返すと、それは「クリーンアップ関数」として扱われます。

```typescript
useEffect(() => {
  const timerId = setInterval(() => console.log('Tick'), 1000);

  // ↓これがクリーンアップ関数
  return () => {
    clearInterval(timerId); // タイマーを解除
  };
}, []);
```

### 実行タイミング
- コンポーネントが画面から消える時（アンマウント時）
- （依存配列がある場合）次回の副作用が実行される直前

### 使用用途
- **イベントリスナーの登録と解除**: `window.addEventListener` と `window.removeEventListener`
- 裏で動き続ける処理など、マウントが解除された後も動き続けては困るものを停止させるために使用します。
- また、API通信などもページ遷移で必要がなくなれば中断することが推奨されています。その際は `AbortController` を使用します。

## 依存配列と参照の同一性
依存配列にオブジェクトや関数を渡すと、再レンダリングのたびに新しいオブジェクト/関数が生成されるため、`useEffect` が意図せず毎回実行されてしまいます（無限ループの原因）。

これを防ぐには、`useMemo` でオブジェクトを、`useCallback` で関数をメモ化（キャッシュ）し、不要な再生成を防ぐ必要があります。

```typescript
const handleClick = useCallback(() => { /* ... */ }, []);
useEffect(() => { /* ... */ }, [handleClick]);
```
